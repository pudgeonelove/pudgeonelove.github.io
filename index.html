Урок 1: Базовые настройки

Что такое переменные?

Переменные — это контейнеры для хранения данных. В промышленном программировании в них часто хранят системные настройки: параметры сервера, версии баз данных или статусы.

server_config.py 
# Настройки локального сервера
server_name = "MainNode-01"
uptime_hours = 1024
is_secure = True

print("Сервер:", server_name)
print("Время непрерывной работы (часы):", uptime_hours)
> Сервер: MainNode-01
> Время непрерывной работы (часы): 1024

Вычисления в коде

Компьютеры созданы для точных вычислений. Давай напишем скрипт, который рассчитает необходимый объем памяти для хранения резервных копий.

storage_calc.py 
# Вводные данные
files_count = 450 # количество файлов
size_per_file = 2.5 # размер в гигабайтах

# Расчет
total_storage = files_count * size_per_file

print("Требуется памяти для бэкапа:", total_storage, "ГБ.")
> Требуется памяти для бэкапа: 1125.0 ГБ.

Ветвления логики (If/Else)

Программа должна уметь принимать решения автономно. Напишем фильтр проверки пароля: если пароль слишком короткий, система откажет в доступе.

auth_check.py 
password = "admin123"
min_length = 12

# Функция len() считает количество символов
if len(password) < min_length:
    print("ОШИБКА БЕЗОПАСНОСТИ: Пароль слишком короткий!")
else:
    print("Доступ разрешен. Добро пожаловать.")
> ОШИБКА БЕЗОПАСНОСТИ: Пароль слишком короткий!

Списки (Lists)

Когда данных много, их неудобно хранить в разных переменных. Списки позволяют хранить целые коллекции — например, очередь важных системных задач.

task_queue.py 
# Массив (список) задач в квадратных скобках
tasks = ["бэкап базы данных", "проверка логов", "обновление антивируса"]

print("Всего задач в очереди:", len(tasks))
# В программировании счет начинается с нуля!
print("Текущая задача:", tasks[0])
> Всего задач в очереди: 3
> Текущая задача: бэкап базы данных

Циклы (For)

Главная сила кода — автоматизация рутины. Цикл for позволяет пройтись по всему списку и выполнить действие с каждым элементом. Напишем скрипт для автоматического опроса датчиков пожарной безопасности.

fire_safety_test.py 
# Список установленных датчиков
sensors = ["Датчик дыма (Холл)", "Температура (Серверная)", "Система вентиляции"]

# Цикл проверяет каждое устройство по очереди
for device in sensors:
    print("Тест связи:", device, "- ОК")
    
print("Все системы пожарной безопасности функционируют штатно.")
> Тест связи: Датчик дыма (Холл) - ОК
> Тест связи: Температура (Серверная) - ОК
> Тест связи: Система вентиляции - ОК
> Все системы пожарной безопасности функционируют штатно.
